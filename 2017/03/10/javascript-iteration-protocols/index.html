迭代协议

发表于 2017-03-10

ES6对ES5进行了补充，新增了两个协议：可迭代协议（The iterable protocol）和迭代器协议（The iterator protocol）。

在搞清楚这两个协议之前，我们先弄清楚这两个概念：可迭代对象（the iterable object）与迭代器（the iterator）。

顾名思义，可迭代对象代表这个对象是可以迭代的，例如可以用for…of循环取值，常见的Array与String都是可迭代对象。那么如何迭代该对象呢？这要求所有可迭代对象都实现一个@@iterator方法，该方法将返回一个迭代器对象，迭代器对象将遵循标准的规则产生一系列的值，所以当使用for…of循环取值时，其实也就是获取可迭代对象的迭代器对象，然后根据这个标准的规则来循环取值而已。

上述中，可迭代对象如何产生一个迭代器这就属于可迭代协议的范畴，而迭代器如何以标准的方式产生一系列的值就属于迭代器协议的范畴。

现在我们来总结一下：

可迭代协议它允许对象定义自己的迭代行为。为了定义该迭代行为，对象必须实现@@iterator方法，也就是说对象必须有一个键值为@@iterator(Symbol.iterator)的成员方法，该方法接收零个参数，返回一个遵循迭代器协议的对象。

Symbol.iterator return iterator

迭代器协议定义了一个标准的方式来产生一系列的值（可以有限也可以无限）。迭代器对象必须实现一个next方法，该方法接收零个参数，返回一个具有两个属性的对象。

next() return { done: boolean, value: value}
当done值为true代表迭代器对象过了迭代序列值的末端，当done值为false代表迭代器对象仍然能够产生下一个值。value值可以是任意的JavaScript对象，当done值为true时，value将被释放。
