Express与Koa对比

发表于 2018-09-16

Express和Koa是Node.js开发中最常见的Web框架，两者的创立者都是tj大神，Express是大哥，显得成熟，Koa是小弟，尤为年轻。

Express原来是基于Connet中间件开发的框架，最新的Express 4.x虽然已经把Connect移除，自己实现了一套Connect的接口，但是核心思想仍然没有变化。此外，Express还自身封装了路由，视图处理等功能。

Koa是基于ES7特性开发的框架，框架本身非常简洁，几乎不包含任何多余功能，一切都借助中间件来解决，同时基于async/await特性的异步流程控制，使其解决了”callback hell”的问题，目前非常受开发者欢迎。

规范的Git代码提交说明

发表于 2018-07-23

更新于2018-10-09 13:59:14

使用Git做代码管理的时候，每次提交都要写Commit Message，否则就不能进行提交。一般来说每次提交的Commit Message都应该说明本次提交的目的，但是事实情况并非如想象的那样。因为没有对提交说明进行规范，每个人的提交说明都可能很随意，可能会出现奇葩不着边际的说明，可能会出现毫无意义的说明，这就违背了Commit Message的初衷。

为了防止这种情况，规范化Commit Message显得很有必要。此外，规范化的Commit Message使得提交历史的提交说明有着特定的格式，在回溯历史的时候能够方便很多，同时它还可以帮助开发者快速的生成Change Log。

目前，开源社区已经有了多种Commit Message的写法规范，本文将介绍其中使用最广的AngularJS’s commit message convention，其也被叫做Conventional Commits。

Docker镜像构建流程

发表于 2018-02-25

构建Docker镜像一共有两种方法：

第一种使用docker commit命令来构建镜像，这种方式和Git的commit有点像，很容易理解，但是这种构建方式相当于制作者暗箱操作，很难记录每一次对镜像的变化，日后很难重制相同的镜像，达不到复用的效果，所以一般不推荐这种构建方式。

第二种是编写一个Dockerfile文件，然后使用docker build命令来构建镜像，其中Dockerfile实际上就是每一步对镜像所做变化的指令集，弥补了第一种方式的缺点，所以这种方式更适合用在生产环境中。
