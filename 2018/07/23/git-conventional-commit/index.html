规范的Git代码提交说明

发表于 2018-07-23

更新于2018-10-09 13:59:14

使用Git做代码管理的时候，每次提交都要写Commit Message，否则就不能进行提交。一般来说每次提交的Commit Message都应该说明本次提交的目的，但是事实情况并非如想象的那样。因为没有对提交说明进行规范，每个人的提交说明都可能很随意，可能会出现奇葩不着边际的说明，可能会出现毫无意义的说明，这就违背了Commit Message的初衷。

为了防止这种情况，规范化Commit Message显得很有必要。此外，规范化的Commit Message使得提交历史的提交说明有着特定的格式，在回溯历史的时候能够方便很多，同时它还可以帮助开发者快速的生成Change Log。

目前，开源社区已经有了多种Commit Message的写法规范，本文将介绍其中使用最广的AngularJS’s commit message convention，其也被叫做Conventional Commits。

Commit Message撰写工具

虽然有了提交说明规范，但是每次提交的时候按照规范来人工书写难免会出现错误，于是需要一个工具来使得开发者每次提交说明都严格遵循规范。

Commitizen是一个Commit Message撰写工具，可以通过npm install -g commitizen安装。

commitizen仅有两个命令:

-   `commitizen init <adapter>`用于安装适配器到项目中
-   `git cz`用于替代`git commit`帮你撰写符合规范的Commit Message并提交

安装完commitizen后在项目运行commitizen init cz-conventional-changelog --save以安装该规范适配器到项目中，然后使用git cz代替git commit命令用于代码提交，这样一个符合要求的Commit Message撰写工具就搭建完成了。

生成CHANGELOG

如果你所有的提交信息都符合规范，那么你就可以根据这些提交信息来自动生成Changelog。

你可以使用conventional-changelog-cli或者standard-version工具帮助你自动生成Changelog。

其中，standard-version是一个更全面的工具，它不仅能够生成CHANGELOG，还能自动完成打标签，提交等一系列工作，并提供灵活性的配置，所以更加推荐读者使用standard-version。
