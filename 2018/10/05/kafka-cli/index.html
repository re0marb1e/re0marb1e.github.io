Kafka入门教程（其二）：Kafka命令行工具

发表于 2018-10-05

上一节我们介绍了 Kafka 的基本安装与配置，这一节我们将通过 Kafka 自带的命令行工具初步体验 Kafka。

Kafka 自带了一系列常用工具，包括内置的 ZooKeeper 启停命令 zookeeper-server-start，zookeeper-server-stop，Kafka的启动停止kafka-server-start，kafka-server-stop，Kafka主题相关命令kafka-topics，Kafka生产者相关命令kafka-console-producer与Kafka消费者相关命令kafka-console-consumer等等。

下面我们来依次介绍Kafka的常用命令行工具：

Kafka 主题相关

我们可以通过 kafka-topics 命令来对 Kafka 主题进行增删改查，对于该命令我们需要指定 ZooKeeper 服务的IP地址与端口号(--zookeeper <String: hosts>)，同时还需要指定是什么操作(--list，--describe，--create，--alter，--delete其中之一)。

  • 创建主题

    创建主题时有两个必填的配置项：

    –replication-factor：备份数，就是数据保存几份，这里设置为3，表示该topic的数据会在3个节点上各保存一份

    –partitions：分区数

  • 修改主题

    –partitions：指明将分区增加至多少个，分区不能减少只能增加。

  • 删除主题

    默认情况下 Kafka 不会真正的删除主题，除非你把 server.properties 中的 delete.topic.enable 置为true。

  • 查看主题列表

    使用该命令可以查看主题列表，你会发现Kafka内部默认就存在一个叫做 __consumer_offsets 的主题。

  • 查看指定主题的信息

你会发现运行 kafka-topics 命令并不要求指定Kafka服务，甚至Kafka服务不启动，kafka-topics 命令也能正常运行。

我们可以通过 kafka-console-producer 进入 Kafka 生产者控制台，此命令必须指定 Broker 列表与需要发布消息的主题名称（即可通过形如kafka-console-producer --broker-list <String: broker-list> --topic <String: topic>的指令进入 Kafaka 生产者控制台）。
