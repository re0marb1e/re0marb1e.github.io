Kafka入门教程（其一）：Kafka基本安装与配置

发表于 2018-08-27

本节主要介绍macOS系统下Kafka的安装与配置，阅读本文之前请确保自己有一定的 Java 知识与 ZooKeeper 知识。

安装

Homebrew

我们将通过 Homebrew 来安装 Kafka ，有关 Homebrew 的详情请查看 Homebrew。

Java

首先，我们需要安装 Java 环境，你可以去官网直接下载 Java SE 的安装包点击运行安装，或者你也可以通过 brew cask 来安装。

ZooKeeper

接下来，我们安装 ZooKeeper ，运行 brew install zookeeper ，安装完成后，你会在 /usr/local/etc/zookeeper/ 目录下看到 ZooKeeper 的配置文件如下所示：

我们打开 zoo.cfg ，找到 clientPort=2181 配置项，通过修改这个可以改变ZooKeeper对外服务的端口号。

至此，整个Kafka的安装流程就完成了。

配置

首先我们启动 ZooKeeper ，你可以直接运行 zkServer start 来启动ZooKeeper，这将使用默认的配置文件 /usr/local/etc/zookeeper/zoo.cfg 启动 ZooKeeper ，然后可以通过 zkServer status 来查看ZooKeeper的运行状态，zkServer stop来停止ZooKeeper的运行。此外，这里推荐你使用 brew services 服务来启动ZooKeeper，brew services 提供一致的命令来管理所有后台服务，可以通过 brew services run | start | stop | restart <formula> 来完成启动停止对应的服务，于是这里我们运行 brew services start zookeeper 来启动ZooKeeper。

当系统提示 ZooKeeper 启动成功后，可以通过 zkCli 来验证 ZooKeeper 服务的可用性，运行 zkCli ，当看到 WatchedEvent state:SyncConnected 的字样时，代表 ZooKeeper 服务运行正常。

接下来我们启动 Kafka，在启动 Kafka 之前我们先来修改一下 Kafka 的配置。默认情况下，Kafka会使用ZooKeeper的根路径作为自己的chroot，考虑到 ZooKeeper 服务存在共享给其他应用使用的可能，所以我们最好配置自己的chroot，这样可以确保不会冲突。于是，我们打开 server.properties 文件，修改配置项 zookeeper.connect 为 zookeeper.connect=localhost:2181/kafka，然后保存文件。

ZooKeeper 和 Kafka 还有很多丰富的配置项，但是作为入门，目前不需要太过关心，现在我们可以启动 Kafka 了。

你可以通过自带的命令 kafka-server-start 与 kafka-server-stop 来启动与停止Kafka，也可以通过我们推荐的 brew services 来启动停止Kafka。

当 Kafka 启动完成后，运行 zkCli 命令进入ZooKeeper控制台，在控制台内运行 ls / ，我们会发现 ZooKeeper 数据结构中多出了 kafka 这个路径，该路径下存放的都是 Kafka 运行所需要的数据。

当完成以上的所有步骤后，你大概了解了 Kafka 的安装与配置，对于 Kafka 命令行的使用请参考Kafka入门教程（其二）：Kafka命令行工具。
