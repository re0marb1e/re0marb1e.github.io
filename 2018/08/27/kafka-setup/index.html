Kafka入门教程（其一）：Kafka基本安装与配置

发表于 2018-08-27

本节主要介绍macOS系统下Kafka的安装与配置，阅读本文之前请确保自己有一定的Java知识与ZooKeeper知识。

安装

我们将通过homebrew来安装Kafka，有关homebrew的详情请查看homebrew

首先，我们需要安装Java环境，你可以去官网直接下载Java SE的安装包点击运行安装，或者你也可以通过brew cask来安装。

接下来，我们安装ZooKeeper，运行brew install zookeeper，安装完成后，你会在/usr/local/etc/zookeeper/目录下看到ZooKeeper的配置文件如下所示：

我们打开zoo.cfg，找到clientPort=2181配置项，通过修改这个可以改变ZooKeeper对外服务的端口号。

至此，整个Kafka的安装流程就完成了。

配置

首先我们启动ZooKeeper，你可以直接运行zkServer start来启动ZooKeeper，这将使用默认的配置（/usr/local/etc/zookeeper/zoo.cfg），然后可以通过zkServer status来查看ZooKeeper的运行状态，zkServer stop来停止ZooKeeper的运行。此外，这里推荐你使用brew services服务来启动ZooKeeper，brew services提供一致的命令来管理所有后台服务，可以通过brew services run | start | stop | restart <formula>来完成启动停止对应的服务，于是这里我们运行brew services start zookeeper来启动ZooKeeper。

当系统提示ZooKeeper启动成功后，可以通过zkCli来验证ZooKeeper服务的可用性，运行zkCli，当看到WatchedEvent state:SyncConnected的字样时，代表ZooKeeper服务运行正常。

接下来我们启动Kafka，在启动Kafka之前我们先来修改一下Kafka的配置。默认情况下，Kafka会使用ZooKeeper的根路径作为自己的chroot，考虑到ZooKeeper服务存在共享给其他应用使用的可能，所以我们最好配置自己的chroot，这样可以确保不会冲突。于是，我们打开server.properties文件，修改配置项zookeeper.connect为zookeeper.connect=localhost:2181/kafka，然后保存文件。

ZooKeeper和Kafka还有很多丰富的配置项，但是作为入门，目前不需要太过关心，现在我们可以启动Kafka了。

你可以通过自带的命令kafka-server-start与kafka-server-stop来启动与停止Kafka，也可以通过我们推荐的brew services来启动停止Kafka。

当Kafka启动完成后，运行zkCli命令进入ZooKeeper控制台，在控制台内运行ls /，我们会发现ZooKeeper数据结构中多出了kafka这个路径，该路径下存放的都是Kafka运行所需要的数据。

当完成以上的所有步骤后，你大概了解了Kafka的安装与配置，对于Kafka命令行的使用请参考下一节(敬请期待)。
