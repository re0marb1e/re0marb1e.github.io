多版本环境管理工具

你可能会遇到一些情况需要在同一个操作系统上安装不同版本的语言环境，比如你想体验一下最新的Java10，同时某些软件只支持Java8，于是你必须同时保留两个环境，同时你还需要能够在操作系统上切换两个版本的Java环境。对于上述问题，很多语言都有相对应的工具来帮助你轻松解决，比如jEnv，rvm等。下面以macOS系统与Zsh为例，整理了各种语言的多版本环境管理工具的介绍以及简单的使用说明。

Java

  3. Java环境添加完成后

    • 通过jenv versions列出所有的Java环境
    • 通过jenv version查看当前的Java环境
    • 通过jenv remove <jdk-version>移除指定的Java环境
    • 通过jenv global <jdk-version>配置全局Java环境
    • 通过jenv local <jdk-version>配置本地Java环境
    • 通过jenv shell <jdk-version>配置当前用户的Java环境

Node.js

  3. 使用nvm安装和管理不同的Node.js环境

    • 通过nvm ls-remote --lts <node-version>查看所有可安装的Node.js版本
    • 通过nvm install <node-version>安装指定的Node.js版本
    • 通过nvm uninstall <node-version>卸载指定的Node.js版本
    • 通过nvm list查看已经安装的Node.js版本
    • 通过nvm current查看当前环境的Node.js版本
    • 通过nvm use <node-version>切换当前环境的Node.js版本

Python

推荐使用pyenv来管理不同的Python环境。

  1. 安装执行(如果之前安装过python，请把之前安装的python卸载再安装pyenv)

  2. 配置pyenv

  3. 使用pyenv安装和管理不同的Python环境

    • 通过pyenv install --list查看所有可安装的Python版本
    • 通过pyenv install <py-version>安装对应的Python版本
    • 通过pyenv unintall <py-version>卸载对应的Python版本
    • 通过pyenv versions查看已经安装的Python版本
    • 通过pyenv version查看当前环境的Python版本
    • 通过pyenv global <py-version>切换全局环境的Python版本
    • 通过pyenv local <py-version>切换局部环境的Python版本(比如当前文件夹下)
    • 通过pyenv shell <py-version>切换当前用户的Python版本，退出后失效

  4. 使用pyenv-virtualenv创建Python虚拟环境
